### 2. Release Floor &amp; Fallback Logic

**Location:** `/home/janfr/dev/pc-switcher/src/pcswitcher/version.py:378-411`

**The get_release_floor() Method:**
```python
def get_release_floor(self, *, include_prereleases: bool = True) -&gt; Release:
    """Return the latest GitHub release that is &lt;= this version."""
    all_releases = get_releases(include_prereleases=include_prereleases)
    candidates = [release for release in all_releases if release.version &lt;= self]
    if not candidates:
        raise RuntimeError(...)
    return max(candidates, key=lambda r: r.version)  # Highest matching release
```

**Fallback in InstallOnTargetJob:**
- **install_on_target.py lines 104-112:** Log warning and call get_release_floor()
- **install.py lines 16, 23:** Additional version check for backward compat with versions &lt; v0.1.0-alpha.4

**Why Exact Version Fails:**
- Dev versions like `0.1.0a5.post272.dev0+d1944d2` are dynamically generated by `uv-dynamic-versioning` based on git history (post-commits, dev suffix)
- They don't correspond to any published GitHub release tag
- Only release tags (e.g., `v0.1.0-alpha.5`) exist as releases and are installable

**Tag Pattern &amp; Release Trigger:**
- Releases appear to be **manual** or **external**
- Tags must follow SemVer format for install script recognition (e.g., `v0.1.0-alpha.5`, `v0.2.0`)
- The install script (`install.sh`) accepts:
  - `VERSION=v0.1.0-alpha.5` (release tag)
  - `--ref main` (branch ref)
  - No version arg → defaults to `main` branch

**Can a Tag on THIS Branch Create an Installable Version?**
- **YES**, with caveat:
  1. Cut a tag on the current branch (e.g., `git tag v0.1.0-alpha.6`)
  2. Push to origin (e.g., `git push origin v0.1.0-alpha.6`)
  3. Tag must be reachable via GitHub API as a release or annotated tag
  4. Run `install.sh` with `VERSION=v0.1.0-alpha.6` or let `curl` fetch the tag
  5. **However:** If the tag is NOT a GitHub Release (created via gh release create), it may not be discoverable by `get_releases()` API call
  6. The install script can still use it directly if pushed as a git tag, but `get_release_floor()` won't find it

**Workflow for Release from Branch:**
- Create tag: `git tag v0.1.0-test-1` on branch HEAD
- Push tag: `git push origin v0.1.0-test-1`
- Create GitHub Release: `gh release create v0.1.0-test-1` (optional but recommended for `get_releases()` discovery)
- Install on target via: `curl ... | VERSION=v0.1.0-test-1 bash`
