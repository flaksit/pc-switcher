## 4. Locked-In Architectural Constraints

**ADR-002 (SSH Communication Channel)**:
- Single persistent SSH connection with multiplexing (`ControlMaster`), not per-operation
- Target-side scripts produce **line-buffered stdout** for progress (no polling)
- Exit codes for success/failure; stdout/stderr separated
- "Forbidden": custom protocols, daemons on target, per-operation SSH connections

**ADR-003 (Python + Task-Specific Languages)**:
- Python orchestrator (Python 3.14, `uv` for dependency management, `asyncssh` library)
- Task-specific bash scripts for individual sync operations (bundled in `scripts/`)
- All orchestration logic in Python; delegation to bash where appropriate

**ADR-005 (Asyncio for Concurrency)**:
- **All Job methods MUST be `async def`** (`validate()`, `execute()`)
- Use `asyncssh` for remote operations (cancellable)
- Use `asyncio.TaskGroup` for structured concurrency
- Signal handlers trigger `asyncio.Task.cancel()` for immediate abort

**ADR-010 (Logging Infrastructure)**:
- stdlib `logging` module with custom Handler
- File logging (JSON), TUI logging (Rich-formatted)
- Configurable log levels separately for file and TUI

## 5. Job Abstraction

**From `/src/pcswitcher/jobs/base.py`:**

```python
class Job(ABC):
    name: ClassVar[str]
    required: ClassVar[bool]
    CONFIG_SCHEMA: ClassVar[dict] = {}
    
    # Async interface
    async def validate(self) -&gt; list[ValidationError]: ...
    async def execute(self) -&gt; None: ...
    
    # Helpers provided to jobs
    @property
    def source(self) -&gt; LocalExecutor: ...  # For local operations
    @property
    def target(self) -&gt; RemoteExecutor: ...  # For remote SSH operations
    
    def _log(host: Host, level: LogLevel, message: str, **extra) -&gt; None
    def _report_progress(update: ProgressUpdate) -&gt; None
    def _validation_error(host: Host, message: str) -&gt; ValidationError
```

**Subclasses:**
- `SystemJob(Job)` — infrastructure (btrfs snapshots, installation); `required=True`, not in config
- `SyncJob(Job)` — user-facing features; `required=False`, configurable via `sync_jobs` in config.yaml
- `BackgroundJob(Job)` — parallel execution (disk monitoring); runs throughout sync

**Every sync job gets**:
- Configuration validation via JSON Schema (declared in `CONFIG_SCHEMA`)
- Logging at 6 levels (DEBUG, FULL, INFO, WARNING, ERROR, CRITICAL)
- Progress reporting with percent/current/total/item
- SSH executors for both source and target
- Graceful cancellation via `asyncio.CancelledError`
