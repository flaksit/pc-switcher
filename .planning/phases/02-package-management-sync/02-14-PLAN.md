Rewrite `docs/adr/adr-020-declarative-package-convergence.md` in place, keeping its `Status: Accepted`, its date, and its existing section skeleton (TL;DR, Implementation Rules, Context, Decision, Consequences, Alternatives Considered, References). This is a correction of a record that was written from decisions later found wrong, not a supersession: do not create ADR-021, and do not add a changelog, a "previously said" note, or any narration of the correction. Write the resulting state as if it had always been the decision.

Six substantive corrections, each traceable to a decision id:

1. Per-manager review (D-24). Delete the entire `### Two-phase convergence and the package-phase coordinator (D-15 + D-24, ADR-014)` subsection and replace it with a subsection recording that each package job runs plan then review then apply inside its own `execute()`; the batching is per manager, grouped by action, never across managers; there is no shared review phase and no coordinator. Record the reason the two decisions reconcile this way: the jobs are independent by D-15, so a cross-manager coordinator contradicts that independence, and the user's own framing was one batched review per manager rather than one prompt for the fleet of managers. In `## Alternatives Considered`, replace the "Per-job self-contained review — rejected" bullet with its inverse: a cross-manager coordinator, rejected because it makes three deliberately independent jobs share a single failure and ordering surface for no user-visible gain.

2. Four jobs (D-15, D-18). Everywhere the ADR says three jobs, say four: `apt_sync`, `snap_sync`, `flatpak_sync`, `manual_installs_sync`. Record that `manual_installs_sync` owns unreproducible detection (both the apt-no-candidate set and the unowned `/usr/local` and `/opt` scan) and the snippet registry, runs its own `dpkg-query` and `apt-cache policy` queries rather than sharing `apt_sync`'s, and carries its own `sync_jobs` enable flag so disabling apt cannot silently disable manual-install detection.

3. Skip-once is a resolution (D-21). Rewrite the `### Mandatory registration and where it terminates (D-21 with D-26 and D-27)` subsection. An unreproducible item ends a run resolved in three ways: it has a snippet, it is recorded machine-specific, or the user chose to skip it once. Only an item nobody decided on leaves the run unclean. Rename the subsection so the word "mandatory" does not appear in its heading.

4. apt collateral classification (D-30). Rewrite `### apt transaction fidelity (D-24, D-25)`. Simulation runs at plan time. Collateral that is auto-installed (absent from the target's `apt-mark showmanual` set) is apt resolving dependencies and proceeds without asking. Collateral that is manually installed becomes its own reviewable item offering install-anyway, skip, or abort. Record why blanket refusal was wrong: it blocks a legitimate install whose only collateral is a dependency nobody chose. Record that the question is asked in the review and never mid-apply.

5. Snippet transport (D-23). Replace every statement that the registry travels via `config_sync` alongside `config.yaml`. Record instead that `manual_installs_sync` pushes `~/.config/pc-switcher/package-snippets.yaml` to the target itself with `send_file()` immediately after its own review, so a snippet authored on the fly during that review reaches the target in the same run, and that `config_sync` carries `config.yaml` only. Record both reasons the alternatives fail: `config_sync` runs before any review, so it cannot carry a snippet the user has not authored yet; and `folder_sync` is a user-controlled job that can be disabled or filtered, so no job's correctness may depend on another job running.

6. Layout, config and documentation (D-31, D-32, D-33). Add three Implementation Rules: package-sync helpers live in a `jobs/packages/` package as `items.py`, `review.py`, `state.py`, `sync_core.py`, while job modules plus `base.py` and `context.py` stay directly in `jobs/` because job discovery resolves a `sync_jobs` key to `jobs/<name>.py`; a job gets a config section only when it has a real key, so `default-config.yaml` and `config-schema.yaml` carry no empty placeholder section; and what a job does is documented in that job's own document, with the configuration reference restricted to configuration.

Then reconcile the `## Implementation Rules` block with all six corrections. Replace the two coordinator rules (the `plan()`/`apply()` rule that requires a coordinator-supplied accepted plan, and the `PackagePhaseCoordinator MUST run every enabled package job's plan() first` rule) with a rule that each package job MUST complete its own review before issuing its own first mutating command. Replace the `Forbidden` bullet that says no job may call its own review/converge sequence inside `execute()` with its inverse: no component outside a package job may own that job's review. Update the snippet-registry rule to name `send_file()` and `manual_installs_sync` as the transport. Update the apt-simulation rule for the auto-versus-manual classification.

Leave untouched, verbatim where possible, every rule and subsection covering the "Do Not Undo" invariants: privileged `/etc/apt` staging under the target user's `~/.cache` promoted with `sudo install`, `send_file` never targeting a path outside the target user's home, transactional repository-group backup and restore around `apt-get update`, no `--delete` mirror of `/etc/apt`, keys copied byte-for-byte and never re-fetched, no `snap refresh --hold`, machine-local decision files never synced, and package databases never rsynced.

Update the `## Context` section: the paragraph currently attributes the coordinator to a cross-AI review finding. Replace it with a plain statement of what Phase 2 must replicate and what the review boundary is, without narrating the review or the correction.

Finally add one Implementation Rule capturing the exit-code invariant already delivered and easy to lose: session status and the CLI exit code MUST derive from `job_results`, never from whether an exception propagated.
