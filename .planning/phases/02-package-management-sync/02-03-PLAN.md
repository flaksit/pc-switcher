## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| package-manager output â†’ command construction | Names and versions parsed from `apt-mark`/`dpkg-query` output are interpolated into subsequent shell commands. |
| source orchestrator â†’ target root | Every converge command runs as root on the target via sudo over SSH (ADR-002, ADR-013). |
| target repositories â†’ target system | The target downloads and installs packages from its own configured repositories (D-28). |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-02-01 | Tampering | `AptSyncJob.converge`, `capture_source_items`, `query_target_items` | high | mitigate | Every interpolated package name goes through `shlex.quote()` before reaching a command string, matching the existing `folder_sync`/`vscode_state_sync` convention (ASVS V5). |
| T-02-04 | Access Control | target install commands | high | mitigate | `validate()` checks `sudo -n true` on the target before any converge runs, so a missing-sudo target fails preflight rather than mid-run with a half-converged system (ASVS V4). |
| T-02-06 | Denial of Service | dpkg frontend lock | medium | mitigate | `validate()` probes `/var/lib/dpkg/lock-frontend` read-only and reports lock contention as a named validation error, so a run colliding with `unattended-upgrades` fails clearly instead of producing N identical opaque per-item errors. |
| T-02-07 | Repudiation | per-item convergence | medium | mitigate | Each item logs its concrete action at FULL and each failure logs its stderr at ERROR, with a per-job INFO summary; a destructive change always leaves an audit trail (ADR-010). |
| T-02-09 | Elevation of Privilege | `apt-get install` invocation | medium | mitigate | `DEBIAN_FRONTEND=noninteractive` and `-y` are set because the executor's `Process` protocol provides no stdin; without them a debconf prompt would hang the sync indefinitely rather than fail. |
| T-02-SC | Tampering | npm/pip/cargo installs | high | mitigate | No new package installs in this plan; `questionary` was gated by the blocking human checkpoint in 02-02. |
</threat_model>
