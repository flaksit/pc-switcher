<objective>
Relocate the package-sync helpers into a `jobs/packages/` package (D-31) and delete the empty per-job config sections (D-32). Both are layout tidy-ups with no behaviour change, done after every behavioural plan has settled so the move is a mechanical rename over the final tree.

Purpose: `jobs/` should contain job modules plus the shared job infrastructure (`base.py`, `context.py`), not a flat pile of `package_*.py` helpers. The helpers become a `jobs/packages/` package with the prefix stripped. Separately, `apt_sync: {}`, `snap_sync: {}` and `flatpak_sync: {}` are empty placeholder sections with banner comments — worse than their absence per D-32, since a job earns a config section only when it has a real key.

Output: `jobs/packages/{items,review,state,sync_core}.py` with every importer updated; the four job modules and `base.py`/`context.py` still directly in `jobs/`; the three empty config sections gone from `default-config.yaml` and `config-schema.yaml`.
</objective>
