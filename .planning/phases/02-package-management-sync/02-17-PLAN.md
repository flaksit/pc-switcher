<objective>
Create `manual_installs_sync`, the fourth package job, owning everything no package manager can reproduce (D-15, D-18), and correct skip-once to be a valid resolution (D-21).

Purpose: unreproducible detection and the snippet registry currently live inside `apt_sync`. Half of what they cover is not apt's business at all — unowned files under `/usr/local` and `/opt` — and folding it into `apt_sync` means disabling apt silently disables manual-install detection with nothing telling the user. D-15 pulls it into its own job with its own enable flag. D-21 corrects a policy the same subsystem enforces: a user who skips an unreproducible item once has made a real decision, and a run where they did so is clean.

Output: a new `ManualInstallsSyncJob` doing its own `dpkg-query`/`apt-cache policy` queries; the unreproducible detection, review-group carve-out, snippet replay, finalize and unresolved-as-failure logic removed from `apt_sync`/the shared base and re-homed on the new job; and `_review_unreproducible_group` counting an explicit skip-once as resolved.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| snippet body to target shell | An opaque snippet is executed on the target; a parsed or templated snippet would be an injection surface |
| decision file to peer | A machine-local decisions file must never cross to a peer, or a machine-specific item propagates |
| detection to review | A detected unreproducible finding must reach the review or be silently dropped |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-02-44 | Tampering | ManualInstallsSyncJob.converge / SnippetRegistry.replay | high | mitigate | Snippet replayed verbatim via `bash -c` with a shlex-quoted body, exit code deciding success; no parsing or templating, so there is no injection beyond what the user authored |
| T-02-45 | Information Disclosure | ManualInstallsSyncJob decision file | high | mitigate | The new job's `manager_id` decision file is covered by the existing non-overridable folder_sync glob exclusion for `*.decisions.yaml`; a unit assertion confirms the new manager's file matches the glob |
| T-02-46 | Repudiation | D-21 skip-once path | medium | mitigate | An explicit skip-once is a recorded decision, distinct in the review outcome from a cancelled prompt, so a clean run is auditable as such |
| T-02-SC | Tampering | npm/pip/cargo installs | high | mitigate | No package installs in this plan; nothing to audit |
</threat_model>
