<objective>
Close the last gap between "packages a package manager can reinstall" and "everything that is actually installed": no-candidate apt packages, and software dropped under `/usr/local` and `/opt` by an install script.

Purpose: the source machine's real inventory has four apt packages with no repo candidate and five unowned trees. Without this, a "successful" sync leaves the target quietly missing them. D-21 makes registration mandatory precisely so the noise D-19 accepts is one-time: a finding either gets a snippet or gets recorded, and then never speaks again.

Output: unreproducible-item detection, a shared snippet registry, and an on-the-fly snippet-capture path inside the review.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user-authored snippet → root shell on the target | An opaque text blob is executed with the target's root privileges. |
| synced snippet registry → every fleet machine | A snippet authored on one machine replays on every other machine. |
| filesystem scan output → command construction | Paths discovered under `/usr/local` and `/opt` are interpolated into subsequent commands. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-02-17 | Elevation of Privilege | snippet replay on the target | medium | accept | Snippets are authored by the human running the sync, during the review, and replay through the same executor as every other converge action. The trust boundary is unchanged: whoever runs pc-switcher already has root on both machines (D-20). No sandboxing is specified by CONTEXT and none is invented here. Canon note: arbitrary-code-execution hardening is covered by `/gsd-secure-phase`; not minted as a bespoke prohibition. |
| T-02-18 | Tampering | snippet replay hang | medium | mitigate | The executor provides no stdin, so an interactive snippet cannot silently wait forever on user input from a terminal that is not attached; the authoring note in the capture prompt states the non-interactive requirement and shows a worked shape. |
| T-02-19 | Information Disclosure | synced snippet registry | low | accept | The registry is deliberately shared (D-23) and lives in the same config the user already syncs; a snippet is install knowledge about a package, not machine-local state. |
| T-02-01 | Tampering | scanned path interpolation | high | mitigate | `shlex.quote()` on every discovered path and on the snippet body before it reaches a command string (ASVS V5). Canon note: injection is canon — covered by `/gsd-secure-phase` and the project's existing quoting convention; not minted as a bespoke prohibition. |
| T-02-SC | Tampering | npm/pip/cargo installs | high | mitigate | No package installs in this plan. |
