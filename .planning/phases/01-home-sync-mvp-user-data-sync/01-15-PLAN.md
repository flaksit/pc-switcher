<objective>
Make the first-sync overwrite warning job-agnostic. Each SyncJob that destructively replaces target state describes its own scope and transport mechanism; the orchestrator assembles the warning from those descriptions instead of reaching into the folder_sync config dict and hardcoding rsync wording.

Purpose: ADR-015 places the "first sync overwrites the target" question at the orchestrator level precisely because it is common to all jobs. Today the orchestrator leaks one job's config shape and transport mechanism into that generic messaging, so a future non-rsync job (packages/docker) added to first-sync scope would inherit an incorrect description. Closes UAT gap 1.

Output: A FirstSyncScope contract, a SyncJob hook, a FolderSyncJob override, and an orchestrator that composes the warning generically.
</objective>
