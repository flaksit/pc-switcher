<objective>
Add the `folder_sync` configuration surface: register it in the config schema (both the `sync_jobs` enable-flag and the top-level job-config object) and ship sensible defaults in `default-config.yaml` — default folders `/home` and `/root` (D-01, D-02, D-03), each as a folder entry with its own exclude list (D-10), and the default machine-specific exclusions (D-11).

Purpose: Without the schema entries the existing `additionalProperties: false` guards reject any config that mentions `folder_sync` (RESEARCH Pitfall 6). The defaults are what every user's `pc-switcher init` writes and what the integration test (plan 06) runs against.

Output: Updated config schema and default config; a real `pc-switcher sync` can discover and configure the folder-sync job.
</objective>
