<objective>
Close the two INFO-level correctness/UX findings outside the divergence guard.

- IN-01 (cli.py): the first-SIGINT cleanup branch awaits `asyncio.wait_for(asyncio.shield(asyncio.sleep(0)), timeout=CLEANUP_TIMEOUT_SECONDS)`, which returns immediately, so it grants no cleanup time and its `TimeoutError` branch is unreachable. Remove the dead construct and make the interrupt messaging truthful.
- IN-02 (orchestrator.py / ui.py): `total_steps = 8 + len(self._config.sync_jobs) + 1` counts every configured job including `enabled: false` ones, while the final step uses only enabled+valid jobs, so the progress bar never reaches 100% in the default config. Set the total from the jobs that will actually run.

Purpose: an honest terminal UX (REQ-terminal-ux) — the progress bar completes and the interrupt prompt does not claim a guarantee it does not keep.

Output: simplified SIGINT cleanup in `cli.py`, a `TerminalUI.set_total_steps` setter, and an orchestrator correction that sets the real total after job discovery.
</objective>
