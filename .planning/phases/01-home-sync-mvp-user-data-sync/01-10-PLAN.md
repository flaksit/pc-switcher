    Create the ADR following the exact structure in ADR-001 (Status, Date, TL;DR, Implementation Rules, Context, Decision, Consequences, References). Status: Accepted. Date: 2026-07-01. Do NOT add a Supersedes header (no divergence ADR exists to supersede).

    The ADR must record these points (write the prose yourself; do not copy this list verbatim):
    - Decision: sync safety = (1) btrfs pre/post snapshots as the rollback backstop, (2) rsync --dry-run whose deletions are itemized in the FULL-level log, (3) a topology-based out-of-order / target-state warn+confirm step that reads the local and target sync-history (last_role + last_peer), and (4) trusting the user. No destructive-preview confirmation by default; no content-based divergence detection.
    - Reject btrfs `find-new` content-detection, with the three verified reasons: it answers the wrong question (measures byte-changes since a generation, not independent user work); on a running desktop normal boot/login/cache churn and unattended upgrades constantly bump generations, forcing an ever-growing exclusion denylist where every exclusion widens the false-NEGATIVE (silent data-loss) window (CR-01 was exactly that); and subvolume granularity does not match the sync scope (/home=@home but /root lives on @).
    - Note the redundancy: rsync `-n --delete` already gives the exact, scope-correct, noise-free preview of what would be destroyed, making content-detection redundant with dry-run + snapshots and a permanent maintenance/safety liability.
    - Implementation Rules (forbidden/required) must state: no `btrfs subvolume find-new` guard; no per-target generation markers; the topology check is warn+confirm (never a hard abort on the legit A→B / work-on-A / A→B pattern, GitHub #159); an explicit `--allow-out-of-order` bypass; deletions persisted at FULL for both dry-run and real runs.
    - References: ADR-013, ADR-014 (and one sentence clarifying that ADR-014's "run divergence/conflict detection during dry-run" is now realized by the topology out-of-order check, not by btrfs find-new — ADR-014 stays immutable and is not superseded), CR-01/CR-02 in 01-VERIFICATION.md, GitHub #159.

    Keep it concise per ADR-001 (Context under ~200 words). Do not leave placeholders.
  </action>
