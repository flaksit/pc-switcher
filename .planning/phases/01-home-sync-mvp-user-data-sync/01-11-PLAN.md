<objective>
Delete the btrfs content-based target-divergence guard from FolderSyncJob and its config-sync helper landmine, keeping the rsync transfer, exclusions, metadata, dry-run, and progress/deletion handling exactly as-is. This resolves CR-01 (unanchored tool-state filter false-negative) and CR-02 (install-artifact false-positive) by removing the code they live in, and fixes WR-01 in config_sync.

Purpose: The content guard measured the wrong thing (bytes changed since a generation, not independent user work) and every exclusion widened a silent data-loss window; safety now comes from snapshots + dry-run deletion log + the topology check (ADR-015). This plan removes the liability without regressing the phase's byte-identity / metadata / exclusion guarantees.

Output: A leaner FolderSyncJob whose only job is the rsync mirror, plus the config_sync prefix-strip fix, plus pruned tests.
</objective>
