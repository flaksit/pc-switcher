<objective>
Create `FolderSyncJob` and implement everything up to (but not including) the rsync transfer: config-driven folder selection, the `validate()` preflight (sudo rsync, acl, folder existence), and the target-divergence guard — the reliability linchpin (D-06/D-07/D-08/D-18). `execute()` is stubbed here and filled by plan 05.

Purpose: The divergence guard is the core safety mechanism that makes mirror-with-`--delete` provably safe. It must abort the sync before any snapshot or transfer when the target has been independently modified since the last sync, while never false-flagging the legitimate A→B-then-A→B-again flow (D-07). Putting it in `validate()` means the orchestrator (Phase 4) aborts cleanly before the destructive mirror.

Scope note: per-file conflict detection is deliberately NOT built (D-09 — the folder/subvolume-granularity guard plus the documented preconditions D-17/D-18/D-19 are sufficient for safety; D-17 single-active-machine and D-19 quiescent-source remain documented user-responsibility assumptions in Phase 1).

Output: A FolderSyncJob whose validate() enforces the preconditions and the divergence guard, with unit tests over mocked executors.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
| -------- | ----------- |
| config → shell (btrfs/rsync preflight) | Folder paths from config are passed into `findmnt`/`btrfs` commands on the target. |
| target filesystem → divergence decision | btrfs generation/find-new output gates whether the destructive mirror is allowed. |
| user → override | `--allow-divergence` lets the user proceed past a detected divergence. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
| --------- | -------- | --------- | -------- | ----------- | --------------- |
| T-04-01 | Tampering (command injection) | folder path / config values in preflight + divergence commands | high | mitigate | Every config-derived path is wrapped with `shlex.quote` before interpolation into a command string; schema (plan 02) constrains values to strings. |
| T-04-02 | Integrity (data loss) | Divergence guard correctness | high | mitigate | File-level find-new detection blocks the mirror when the target changed; first-sync/None and snapshot-bump cases are handled so the guard neither false-negatives (unguarded delete) nor false-positives (blocks legit flow). Unit tests cover all branches; plan 06 validates on real VMs. |
| T-04-02b | Repudiation | Override usage | medium | mitigate | A dry-run/override that proceeds past divergence is logged at WARNING with the folder path — an audit trail in the JSON log. |
| T-04-03 | Denial of service | acl missing silently degrades -A | medium | mitigate | validate() fails fast if the acl package is absent on either end (RESEARCH Pitfall 5), so metadata loss is caught before sync, not after. |
| T-04-SC | Tampering | npm/pip/cargo installs | low | accept | No packages installed; RESEARCH Package Legitimacy Audit confirms no new dependencies. |
</threat_model>
