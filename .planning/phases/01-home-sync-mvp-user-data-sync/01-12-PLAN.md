<objective>
Simplify sync-history to the topology model: add last_peer alongside last_role, provide a reader and a pure JSON parser for (role, peer), and delete the btrfs per-target generation store that the removed divergence guard depended on.

Purpose: The orchestrator's new out-of-order / target-state check (plan 01-13) needs to know, for both this machine and the target, who each last synced with and in what role. The generation store existed only for the content guard that plan 01-11 removed.

Output: A sync_history module whose schema is roughly {last_role, last_peer}, plus matching tests.
</objective>
