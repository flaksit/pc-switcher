<objective>
Give a user-declined confirmation its own outcome so it is logged once at WARNING (not CRITICAL) and surfaced once by the CLI (not twice). Introduce a SyncAbortedByUser exception raised at the decline sites, caught distinctly by both the orchestrator and the CLI.

Purpose: Today declining a prompt raises a plain RuntimeError that falls into the generic `except Exception` handler (orchestrator logs every exception at CRITICAL and re-raises; the CLI outer handler then prints the same "Sync failed" text again). CRITICAL is documented as "unrecoverable, sync must abort" (models.py) — but a user answering "no" is expected control flow, not a failure. Closes UAT gap 2.

Output: A dedicated abort exception + status, distinct handling in run() and the CLI, and tests proving one WARNING log and one calm message.
</objective>
